For Process Documenters

How to Capture a Process

1. Open the Capture Tool
  • Visit the website URL provided to you, OR
  • Double-click capture.html if you downloaded it
2. Start Recording
  • Click "Start Recording"
  • A dialog will appear asking what to share
  • Choose what to capture:
    • Application Window - Select QuickBooks, Excel, or any desktop app (recommended)
    • Entire Screen - Captures everything on your monitor
    • Browser Tab - For web applications only
  • Click "Share"
💡 Works with ANY software:
  • Desktop applications (QuickBooks, SAP, Excel, Photoshop, AutoCAD, etc.)
  • Web applications (Salesforce, HubSpot, Google Workspace, etc.)
  • Multiple applications (switch between apps during recording)
  • See CAPTURE_CAPABILITIES for full details
3. Capture Your Steps
  • Perform your task normally
  • After each important action, click "Capture Step"
  • The tool will screenshot what you just did
4. Add Descriptions
  • Type a description for each step
  • Explain what you did and why
  • Be specific (e.g., "Click the blue 'Submit' button in the top right")
5. Organize Your Steps
  • Use ⬆️ and ⬇️ buttons to reorder steps
  • Delete unnecessary steps with delete button
  • Add manual steps with "Add Manual Step"
6. Export Your Work

Option A: For Process Analysis

  • Click "Export for Process Analysis"
  • Save the .json file
  • Send it to your analyst/manager

Option B: For Immediate Sharing

  • Click "Export as HTML" for a webpage
  • Click "Export as Markdown" for documentation
  • Click "Print / Save as PDF" for printing

For Analysts (Analysis Features)

How to Analyze a Process

1. Get the Process File

Receive the .json file from the documenter

2. Open the Analysis Tool
  • Download analyze.html to your computer
  • Double-click to open in your browser
  • (Keep this file local - don't host it online)
3. Import the Process
  • Click "Import Process"
  • Select the .json file
  • All steps will load automatically
4. Run Process Analysis
  • Click "Analyze Process"
  • Wait 10-30 seconds
  • System will analyze the entire workflow
5. Review Analysis Insights

The analysis panel shows:

  • Efficiency Score (0-100) - How optimized the process is
  • Key Inefficiencies - Problems in the workflow
  • Optimization Suggestions - How to improve
  • Automation Opportunities - Steps that could be automated
  • Step Notes - Specific feedback on each step
6. Export with Insights
  • Click "Export as HTML" - Includes analysis insights
  • Click "Export as Markdown" - For documentation
  • All exports include the analysis insights

💡 Pro Tips

For Better Captures

  • Capture each distinct action as a separate step
  • Write clear descriptions (imagine you're teaching someone new)
  • Include screenshots of important screens
  • Use the title field to name your guide clearly
  • Don't capture too many steps at once (break into smaller guides)

For Better Analysis

  • Make sure descriptions are detailed before exporting
  • Review analysis suggestions carefully - they're recommendations, not rules
  • Combine analysis insights with your domain knowledge
  • Look for patterns across multiple process analyses

🎨 Customization

Change Guide Title

  • Click the title field at the top
  • Type your new title
  • Examples: "How to Create a New Customer Account", "Onboarding Checklist"

Add Images Manually

  • Click "Upload Image" on any step
  • Select an image from your computer
  • Works great for adding diagrams or additional screenshots

Reorder Steps

  • Use ⬆️ to move a step up
  • Use ⬇️ to move a step down
  • Keep reorganizing until the flow makes sense

📤 Export Formats Explained

JSON (For Process Analysis)

  • Small file size
  • Includes all data
  • Can be imported back
  • Not human-readable
Use when: Sending to someone for process analysis

HTML (Webpage)

  • Opens in any browser
  • Looks professional
  • Can be shared easily
  • Includes images
Use when: Sharing final documentation

Markdown (.md)

  • Works with documentation systems
  • Easy to edit
  • Version control friendly
  • Works in wikis, Notion, etc.
Use when: Creating technical documentation

PDF (Print)

  • Universal format
  • Can't be edited
  • Professional appearance
  • Easy to email
Use when: Creating formal documentation or printing

🔒 Privacy & Data

What Gets Saved?

  • Screenshots you capture
  • Text descriptions you write
  • Step order and organization

Where Is Data Stored?

  • In your browser only while you're working
  • In export files when you export
  • Never sent to any server (except process analysis text)

Process Analysis Privacy

  • Only step descriptions are sent to the analysis engine
  • Screenshots are NOT sent unless you choose to export them
  • Analysis happens in real-time and isn't stored

❓ FAQ

Q: Can I capture desktop applications like QuickBooks, Excel, or SAP?

A: Yes! When you start recording, choose "Application Window" and select any desktop app. This works with all installed software, not just web browsers.

Q: Do I need to install anything?

A: No! Just open the HTML file in a modern web browser.

Q: Does it work offline?

A: Capture tool works offline, process analysis requires internet.

Q: Can I edit steps after capturing?

A: Yes! Edit descriptions, add images, reorder, or delete steps anytime.

Q: How many steps can I capture?

A: Unlimited, but we recommend keeping guides under 20 steps for clarity.

Q: Can I pause recording?

A: Yes, just don't click "Capture Step" until you're ready. Or stop and resume later.

Q: Does it work on mobile?

A: Screen recording works best on desktop. You can view/edit on mobile.

🆘 Common Issues

"Screen sharing not working"

  • Grant permission when prompted
  • Use Chrome, Firefox, or Edge
  • Make sure the site uses https://

"Can't import JSON file"

  • Make sure file ends in .json
  • File must be exported from capture tool
  • Try re-exporting

"Process analysis failed"

  • Make sure you have internet connection
  • Try refreshing the page
  • Check that you're using analyze.html locally

Need more help? Check the README or contact your system administrator.
