**Perfect for:**
- Training materials
- Standard operating procedures (SOPs)
- Software tutorials
- Business process documentation
- Onboarding guides

## Recording Your First Process

### Example: Documenting How to Create an Invoice

Let's walk through a complete example.

#### Step 1: Prepare

1. **Close unnecessary applications**
   - Fewer apps = faster performance
   - Less clutter in screenshots

2. **Open the app you'll document**
   - Example: Excel, QuickBooks, etc.
   - Have it ready before starting

3. **Launch DocuCapture**
   - From Desktop or Start Menu

#### Step 2: Start Recording

1. **Click "Start Recording"**
   
2. **Select what to capture:**
   - **Entire Screen** - Captures everything
   - **Specific Window** - Just one application
   
   ğŸ’¡ **Tip:** Select specific window to avoid showing personal info

3. **Click the window/screen** to confirm

4. **Floating window appears** in bottom-right

#### Step 3: Perform Your Process

Now work normally in your target application.

**Example: Creating Invoice in Excel**

1. **Open Excel** (if not already open)

2. **Click File â†’ New**
   - Press `Ctrl + Shift + S` to capture
   - Floating window flashes
   - Continue working!

3. **Select "Blank Workbook"**
   - Press `Ctrl + Shift + S`

4. **Type "INVOICE" in cell A1**
   - Press `Ctrl + Shift + S`

5. **Format header cells**
   - Press `Ctrl + Shift + S`

6. **Enter customer information**
   - Press `Ctrl + Shift + S`

Continue for each major action...

#### Step 4: Stop Recording

When finished:
- Click "Stop Recording" on floating window
- Or press `Ctrl + Shift + R`
- Floating window disappears

#### Step 5: Review and Edit

DocuCapture window shows all your captured steps!

1. **Click each step**
2. **Add descriptions:**
   - Step 1: "Open Excel"
   - Step 2: "Create new blank workbook"
   - Step 3: "Add invoice title"
   - etc.

3. **Delete unwanted steps:**
   - Click "Delete" button
   - Remove mistakes or duplicates

#### Step 6: Export

1. **Click "Export as HTML"**
2. **Choose location** (e.g., Documents)
3. **Name your file:** "How to Create Invoice"
4. **Click "Save"**

**Done!** You have professional documentation! ğŸ‰

## Capturing Steps

### Two Ways to Capture

#### Method 1: Keyboard Shortcut (Recommended)

- Press `Ctrl + Shift + S` anytime
- Works from ANY application
- Even when DocuCapture is minimized
- Fastest method!

**When to use:**
- Documenting multi-app workflows
- Quick captures
- Hands stay on keyboard

#### Method 2: Click Button

- Click "Capture Step" on floating window
- Or click "Capture Step" in main window
- Works when recording is active

**When to use:**
- Single-application documentation
- Mouse-heavy processes
- Demo/training situations

### What Gets Captured

Each capture includes:
- **Screenshot** - Current screen content
- **Timestamp** - When it was captured
- **Step number** - Automatic numbering
- **Description field** - For you to fill

### Best Practices for Capturing

#### Capture At Key Moments

**DO capture:**
- âœ… Opening an application
- âœ… Clicking important buttons
- âœ… Entering critical data
- âœ… Completing a section
- âœ… Final result/output

**DON'T capture:**
- âŒ Every tiny mouse movement
- âŒ Typing individual letters
- âŒ Menu hovering
- âŒ Mistakes you'll undo

**Rule of thumb:** Capture 5-15 steps for most processes

#### Timing Your Captures

**Before or After?**

Capture **AFTER** completing the action:
- Type the data â†’ Capture
- Click the button â†’ Capture
- Complete the form â†’ Capture

This shows the **result** of each action, which is clearer for viewers.

#### Screen Coverage

**Full Screen Captures:**
- Shows complete context
- Good for overview steps
- Includes application UI

**Window Captures:**
- Focuses on specific app
- Cleaner screenshots
- Hides desktop clutter

Choose based on your audience and purpose.

## Editing and Organizing

### Adding Descriptions

**Why descriptions matter:**
- Screenshots alone aren't enough
- Explain the "why" not just "what"
- Help viewers understand context

**Good descriptions:**
- âœ… "Click File â†’ New to create a blank workbook"
- âœ… "Enter customer name in the 'Bill To' field"
- âœ… "Select 'Save as PDF' from the dropdown menu"

**Poor descriptions:**
- âŒ "Click button"
- âŒ "Type stuff"
- âŒ "Next step"

**Template:**
```
[Action] + [Location] + [Purpose]
Example: "Click 'Submit' button to send the form"
```

### Reordering Steps

#### Drag and Drop (Timeline View)

1. **Switch to Timeline View**
   - Click "Timeline View" button
   - Or press `Ctrl + Shift + T`

2. **Drag steps to reorder**
   - Click and hold a step thumbnail
   - Drag left or right
   - Drop in new position

3. **Steps automatically renumber**

#### Using Arrows (List View)

1. **Stay in List View** (default)

2. **Click arrow buttons**
   - â†‘ Move step up
   - â†“ Move step down

### Deleting Steps

**Remove unwanted steps:**

1. **Click the step** to select
2. **Click "Delete" button**
3. **Confirm deletion**
4. **Steps automatically renumber**

**Common reasons to delete:**
- Captured mistake
- Duplicate capture
- Irrelevant screen
- Accidentally hit `Ctrl + Shift + S`

### Adding Manual Steps

**Sometimes you need steps without screenshots:**

1. **Click "Add Manual Step"**

2. **Enter description:**
   - "Wait 5 minutes for processing"
   - "Call customer to confirm"
   - "Review output for accuracy"

3. **Manual steps have no screenshot**
   - Show as text only
   - Useful for non-visual actions

### Uploading Images

**Add custom images to steps:**

1. **Click a step**
2. **Click "Upload Image"**
3. **Select image file**
4. **Image replaces screenshot**

**Use cases:**
- Better quality diagram
- Annotated screenshot
- Clearer example

**To use:**

4. **Groups appear automatically**

**Example:**

**Before grouping:**
- Step 1: Open Excel
- Step 2: Create new workbook
- Step 3: Enter customer name
- Step 4: Enter customer address

- **Setup Phase**

  - Step 2: Create new workbook

  - Step 3: Enter customer name
  - Step 4: Enter customer address
- **Invoice Details**
  - Step 5: Add invoice items
  - Step 6: Calculate totals
- **Final Steps**
  - Step 7: Save file

**Benefits:**
- Better organization

### Redaction Tools

**Hide sensitive information in screenshots:**

#### When to Use

Redact these types of data:
- Personal information (names, addresses)
- Financial data (account numbers, amounts)
- Passwords and credentials
- Confidential business data
- Private emails or messages

#### How to Redact

**Method 1: Auto-Detect**

1. **Click a step with screenshot**
2. **Click "Redact Data"**
3. **Click "Auto-Detect Sensitive Data"**
4. **DocuCapture finds:**
   - Social Security Numbers
   - Credit card numbers
   - Email addresses
   - Phone numbers
   - Common password patterns

5. **Review and apply**

**Method 2: Manual Selection**

1. **Click "Redact Data"**
2. **Click and drag** to select area
3. **Choose redaction method:**
   - **Blur** - Blurs the area
   - **Pixelate** - Pixelates content
   - **Black Box** - Solid black rectangle

4. **Click "Apply"**

**Multiple redactions:**
- Create as many as needed
- Each screenshot can have multiple redacted areas
- All saved with the step

## Exporting Your Documentation

### Export Formats

#### HTML (Web Page)

**Best for:**
- Online documentation
- Company wikis
- Knowledge bases
- Easy sharing via email

**Features:**
- Responsive design
- Works in any browser
- Clean, professional layout
- Images embedded
- No software needed to view

**How to export:**
1. Click "Export as HTML"
2. Choose save location
3. Name your file
4. Click "Save"

**Result:**
- Single .html file
- Open in Chrome, Edge, Firefox, etc.
- Can upload to website
- Share via email/Teams/Slack

#### Markdown

**Best for:**
- GitHub documentation
- Technical documentation
- Version control
- Plain text editing

**Features:**
- Plain text format
- Easy to edit
- Git-friendly
- Can convert to other formats

**How to export:**
1. Click "Export as Markdown"
2. Choose save location
3. Name your file (ends in .md)
4. Click "Save"

**Result:**
- Text file with markdown syntax
- Images saved separately
- Open in any text editor
- Preview on GitHub

#### PDF

**Best for:**
- Printed documentation
- Official documents
- Archival purposes
- Universal sharing

**Features:**
- Professional appearance
- Can't be edited (secure)
- Prints well
- Universal format

**How to export:**
1. Click "Export as HTML" first
2. Open the HTML file
3. Press `Ctrl + P` (Print)
4. Select "Save as PDF"
5. Click "Save"

**Result:**
- PDF file
- Open in Adobe Reader, browsers
- Print or share

#### JSON (Analysis)

**Best for:**
- Data analysis
- Process mining
- Integration with other tools
- Programmatic access

**Features:**
- Structured data format
- Machine-readable
- Includes all metadata
- Timestamps included

**How to export:**
1. Click "Export for Analysis"
2. Or press `Ctrl + Shift + A`
3. Choose save location
4. Save as .json file

**Contains:**
- All step descriptions
- Screenshots (base64)
- Timestamps
- Grouping information
- Redaction data

## Tips and Best Practices

### Planning Your Documentation

**Before you start:**

1. **Know your audience**
   - Beginners? Include more detail
   - Experts? Focus on key steps

2. **Define your goal**
   - Training new employees?
   - Creating SOP?
   - Troubleshooting guide?

3. **Prepare your environment**
   - Clean desktop
   - Close unnecessary apps
   - Test the process first

### Capturing Strategy

#### The "Narrative" Approach

**Tell a story with your steps:**

1. **Introduction**
   - "How to create monthly sales report"
   - Capture dashboard/starting point

2. **Setup**
   - Opening applications
   - Logging in
   - Navigating to tools

3. **Main Process**
   - Step-by-step actions
   - One screenshot per major action

4. **Completion**
   - Final result
   - Success confirmation
   - Next steps

#### The "Reference" Approach

**Quick lookup guide:**

1. **No introduction needed**
2. **Direct to key actions**
3. **Minimal descriptions**
4. **Focus on results**

Choose based on your documentation purpose.

## Troubleshooting

### Common Issues and Solutions

#### Floating Window Not Visible

**Symptoms:**
- Started recording
- Don't see floating button
- Can't capture steps

**Solutions:**
1. Check bottom-right corner of screen
2. Minimize other windows
3. Check if recording actually started (recording indicator)
4. Restart DocuCapture
5. Try "Run as administrator"

---

#### Screenshots Are Blurry

**Symptoms:**
- Images look fuzzy
- Text hard to read
- Poor quality

**Solutions:**
1. Use higher resolution display settings
2. Capture specific window instead of full screen
3. Don't resize browser/application during capture
4. Check display scaling (Windows Settings)

---

#### Keyboard Shortcuts Don't Work

**Symptoms:**
- Press `Ctrl + Shift + S`
- Nothing happens
- No capture

**Solutions:**
1. Verify recording is active
2. Check if another program uses same shortcut
3. Close other screen recording software
4. Run DocuCapture as administrator
5. On some keyboards, try `Win + Shift + S` instead

---

#### Can't Export to PDF

**Symptoms:**
- No PDF export button
- Export fails

**Solution:**
1. Export as HTML first
2. Open HTML in browser
3. Use browser's "Print to PDF" feature:
   - Press `Ctrl + P`
   - Select "Microsoft Print to PDF"
   - Click "Print"
   - Choose save location

---

#### App Running Slowly

**Symptoms:**
- Laggy interface
- Slow captures
- Freezing

**Solutions:**
1. Close other applications
2. Reduce number of steps (break into smaller sessions)
3. Check RAM usage in Task Manager
4. Restart computer
5. Capture lower resolution screen

---

#### Exported HTML Won't Open

**Symptoms:**
- Double-click HTML file
- Error message appears
- Page doesn't load

**Solutions:**
1. Right-click HTML file
2. Select "Open with"
3. Choose your browser (Chrome, Edge, Firefox)
4. If images missing: Keep HTML file with associated folder
