Getting Started

DocuCapture has two tools that work together:

1
Capture Tool (Remote) Use this tool to record your process and capture screenshots. Works in any browser. Deploy online for team access.
2
Process Analyst (Local) Import your captured process and get optimization insights. Run this locally for security and privacy.
💡 Pro Tip

Host the Capture Tool online (GitHub Pages is free!) so your team can access it anywhere. Keep the Analyst tool downloaded on your computer.

3

Starting a Recording

Let's capture your first process! Here's what you'll see:

Capture Tool Controls

1
Click "Start Recording" Your browser will ask what to share
2
Choose what to capture Select: Application Window (recommended), Entire Screen, or Browser Tab
3
Select your application Pick QuickBooks, Excel, SAP, or any other application
4
Click "Share" Recording begins! You'll see a red recording indicator.
4

Capturing Steps

As you perform your process, capture screenshots at each important action:

Recording Active
Click "Capture Step" after each action
1
Perform an action Example: Click the "New Customer" button in QuickBooks
2
Click "Capture Step" Screenshot is automatically taken
3
Add description Type what you just did: "Click New Customer button in top menu"
4
Repeat Continue performing actions and capturing steps
💡 Best Practice

Capture one distinct action per step. Instead of "Enter customer details", create separate steps for name, address, phone, etc.

Protecting Sensitive Data

DocuCapture automatically detects sensitive information and alerts you to redact it:

⚠ Sensitive Data Detected!

Possible password field detected, Possible account number visible

When you see this alert, click the "Redact Data" button to protect sensitive information:

Choose Redaction Style:

Username: john.doe@company.com
Password: ••••••••
Account #: 12345-67890
1
Auto-Detection System scans for passwords, credit cards, SSNs, emails, API keys
2
Visual Alert Red border and warning message when sensitive data found
3
Click & Drag to Redact Select sensitive areas with your mouse
4
Apply & Save Redactions are permanently applied to the screenshot
6

Exporting Your Documentation

Once you've captured all your steps, export your guide in multiple formats:

Export Options

1
Export as HTML Standalone webpage - perfect for sharing via email or intranet
2
Export as Markdown Great for documentation systems, GitHub, Notion
3
Export for Analysis Creates JSON file to import into Process Analyst tool
4
Print / Save as PDF Professional PDF for formal documentation
💡 Recommended Workflow

1. Export for Analysis first → 2. Optimize with Process Analyst → 3. Export final documentation as HTML or PDF
